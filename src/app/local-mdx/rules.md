- 구조
  질문이나 의문에서 시작해서 → 탐구 → 발견 → 결론 순서로 흐를 것
  각 섹션이 독립적으로 읽혀도 되지만, 앞 섹션의 논점을 다음 섹션이 이어받을 것

- 문체
  감정이나 깨달음의 과정을 중계하지 말고, 관찰한 것을 바로 독자에게 던질 것
  수사적 질문보다 직접 단언. "이게 맞는 건가?" 대신 "이건 아니었다"
  "생각해보면", "그제야", "이상하게도" 같은 독백 신호어를 피할 것

- 예시
  코드나 구체적 사례가 설명보다 먼저. 설명은 예시 뒤에서 의미를 정리하는 역할
  예시가 여러 개면 각각 소제목으로 나눠서 독립된 케이스로 읽히게 할 것

- 분량 감각
  핵심 논점에서 벗어나는 관찰은 글이 길어져도 과감하게 뺄 것
